# REGRAS DE CRIAÇÃO DE ID DOS ALUNOS

Baseado nos dados do arquivo alunos_ativos.json, este documento estabelece as regras
padronizadas para criação de IDs únicos dos alunos no sistema.

## ESTRUTURA DO ID

O ID do aluno é uma string de exatamente 15 caracteres que segue o padrão:
STD_[INICIAIS_INTERCALADAS_COM_DÍGITOS]_[CURSO]

## COMPONENTES DO ID

### 1. Prefixo Padrão
- Formato: STD_
- Função: Identificação padronizada no banco de dados indicando que se trata de um registro de aluno
- Tamanho: 4 caracteres fixos

### 2. Seção Central (Iniciais Intercaladas)
- Regra: Alternar entre iniciais do nome e dígitos de preenchimento
- Iniciais do Nome: Extrair as primeiras letras de cada palavra do nome do aluno
- Exceções: Ignorar preposições e artigos como "da", "de", "do", "dos", "das"
- Dígitos de Preenchimento: Na seguinte ordem de prioridade:
  1. 3 últimos dígitos do CEP (removendo hífen)
  2. 3 últimos dígitos do celular
  3. Caracteres do email (conforme necessário)

### 3. Sufixo do Curso
- Formato: _[INICIAL][NÚMERO]
- Função: Identificar o curso e nível do aluno
- Exemplos:
  - KIDS 1 → _K1
  - SEEDS 1 → _S1
  - TEENS 1 → _T1
  - TWEENS 1 → _TW1

## ALGORITMO DE INTERCALAÇÃO

1. Extrair iniciais do nome (ignorando preposições)
2. Obter dígitos de preenchimento (CEP + celular + email)
3. Calcular espaço disponível para intercalação (15 - 4 - tamanho_sufixo)
4. Priorizar iniciais: usar TODAS as iniciais disponíveis primeiro
5. Intercalar: inicial → dígito → inicial → dígito → ...
6. Se houver mais iniciais que espaços ímpares, usar todas as iniciais consecutivamente
7. Preencher espaços restantes com dígitos conforme necessário

## EXEMPLOS PRÁTICOS

### Exemplo 1: Benjamin Sousa
Dados:
- Nome: "Benjamin Sousa"
- Curso: "KIDS 1"
- Email: "amandahellen@gmail.com"
- Celular: "98992253261"
- CEP: "65063-903"

Processo de criação:
1. Prefixo: STD_
2. Iniciais: Benjamin Sousa → B, S
3. Dígitos de preenchimento:
   - CEP: 65063-903 → últimos 3 dígitos: 903
   - Celular: 98992253261 → últimos 3 dígitos: 261
   - Total de dígitos: 903261
4. Sufixo do curso: KIDS 1 → _K1
5. Estrutura base: STD_ + intercalação + _K1
6. Espaço para intercalação: 15 - 4 - 3 = 8 caracteres
7. Intercalação: B + 9 + S + 0 + 3 + 2 + 6 + 1 = B9S03261

ID Final: STD_B9S03261_K1

### Exemplo 2: Layla Kahanna da Costa Nogueira
Dados:
- Nome: "Layla Kahanna da Costa Nogueira"
- Curso: "SEEDS 1"

Processo de criação:
1. Prefixo: STD_
2. Iniciais: Layla Kahanna [da] Costa Nogueira → L, K, C, N (ignorando "da")
3. Dígitos de preenchimento:
   - CEP: 65056-656 → últimos 3 dígitos: 656
   - Celular: 98989180860 → últimos 3 dígitos: 860

6. Espaço para intercalação: 15 - 4 - 3 = 8 caracteres

ID Final: STD_L6K5C6N0_S1

### Exemplo 3: Maria Antonia de Sousa Ramos Silva (Caso Especial)
Dados:
- Nome: "Maria Antonia de Sousa Ramos Silva"
- Curso: "TEENS 1"

## REGRAS DE INTERCALAÇÃO DETALHADAS

### Padrão de Alternância Atualizado
- **PRIORIDADE 1:** Usar TODAS as iniciais do nome (obrigatórias)
- **PRIORIDADE 2:** Intercalar com dígitos nos espaços restantes
- Posição ímpar: Inicial do nome (1ª, 3ª, 5ª, 7ª posição)
- Posição par: Dígito de preenchimento (2ª, 4ª, 6ª, 8ª posição)

### Ordem de Prioridade dos Dígitos
1. CEP: Usar os 3 últimos dígitos primeiro
2. Celular: Usar os 3 últimos dígitos em seguida
3. Email: Usar caracteres numéricos do email se necessário

### Tratamento de Casos Especiais
- **Iniciais obrigatórias:** TODAS as iniciais devem ser incluídas no ID
- Se houver mais iniciais que espaços ímpares, usar iniciais consecutivamente
- Dígitos são usados apenas para completar os 15 caracteres
- Se não houver dígitos suficientes, usar '0' como preenchimento
- Se CEP estiver vazio, pular para próxima fonte de dígitos

## CÓDIGOS DE CURSOS
- KIDS: K + número do nível
- SEEDS: S + número do nível
- TEENS: T + número do nível
- TWEENS: TW + número do nível
- ADVANCED: A + número do nível
- KINDER: KD + código específico

## VALIDAÇÃO FINAL
- O ID final deve ter exatamente 15 caracteres
- Formato: STD_XXXXXXXX_XX
- Todos os caracteres alfabéticos devem estar em MAIÚSCULO
- Não deve conter espaços ou caracteres especiais (exceto underscore)

## BENEFÍCIOS DO SISTEMA

1. Padronização: Identificação consistente no banco de dados
2. Rastreabilidade: Informações do aluno codificadas no próprio ID
3. Unicidade: Combinação de dados pessoais garante IDs únicos
4. Legibilidade: Estrutura clara e interpretável com intercalação organizada

Este sistema garante que cada aluno tenha um identificador único, padronizado e
informativo, com a intercalação de iniciais e dígitos proporcionando maior
variabilidade e unicidade dos IDs gerados.
