# REGRAS DE CRIAÇÃO DE ID DOS FUNCIONÁRIOS

Baseado nos dados do arquivo dados_funcionarios.json, este documento estabelece as regras
padronizadas para criação de IDs únicos dos funcionários no sistema.

## ESTRUTURA DO ID

O ID do funcionário é uma string de exatamente 15 caracteres que segue o padrão:
EMP_[INICIAIS_INTERCALADAS_COM_DÍGITOS]_[CARGO]

## COMPONENTES DO ID

### 1. Prefixo Padrão
- Formato: EMP_
- Função: Identificação padronizada no banco de dados indicando que se trata de um registro de funcionário
- Tamanho: 4 caracteres fixos

### 2. Seção Central (Iniciais Intercaladas)
- Regra: Alternar entre iniciais do nome e dígitos de preenchimento
- Iniciais do Nome: Extrair as primeiras letras de cada palavra do nome do funcionário
- Exceções: Ignorar preposições e artigos como "da", "de", "do", "dos", "das"
- Dígitos de Preenchimento: Na seguinte ordem de prioridade:
  1. 3 últimos dígitos do CEP (removendo hífen)
  2. 3 últimos dígitos do telefone

### 3. Sufixo do Cargo
- Formato: _[CÓDIGO_CARGO]
- Função: Identificar o cargo do funcionário
- Códigos dos Cargos:
  - ADM. PEDAGÓGICO → _AP
  - CDA → _CDA
  - PROFESSORA → _PR
  - PROFESSOR → _PR
  - CONSULTOR COMERCIAL → _CC
  - ADM. FINANCEIRO → _AF
  - COORD. PEDAGÓGICA → _CP

## ALGORITMO DE INTERCALAÇÃO

1. Extrair iniciais do nome (ignorando preposições)
2. Obter dígitos de preenchimento (CEP + telefone)
3. Calcular espaço disponível para intercalação (15 - 4 - tamanho_sufixo)
4. Priorizar iniciais: usar TODAS as iniciais disponíveis primeiro
5. Intercalar: inicial → dígito → inicial → dígito → ...
6. Se houver mais iniciais que espaços ímpares, usar todas as iniciais consecutivamente
7. Preencher espaços restantes com dígitos conforme necessário

## EXEMPLOS PRÁTICOS

### Exemplo 1: Aimeê Almeida de Oliveira
Dados:
- Nome: "Aimeê Almeida de Oliveira"
- Cargo: "ADM. PEDAGÓGICO"
- CEP: "65110-000"
- Telefone: "98991075072"

Processo de criação:
1. Prefixo: EMP_
2. Iniciais: Aimeê Almeida [de] Oliveira → A, A, O (ignorando "de")
3. Dígitos de preenchimento:
   - CEP: 65110-000 → últimos 3 dígitos: 000
   - Telefone: 98991075072 → últimos 3 dígitos: 072
   - Total de dígitos: 000072
4. Sufixo do cargo: ADM. PEDAGÓGICO → _AP
5. Estrutura base: EMP_ + intercalação + _AP
6. Espaço para intercalação: 15 - 4 - 3 = 8 caracteres
7. Intercalação: A + 0 + A + 0 + O + 0 + 7 + 2 = A0A0O072

ID Final: EMP_A0A0O072_AP

## REGRAS DE INTERCALAÇÃO DETALHADAS

### Padrão de Alternância
- **PRIORIDADE 1:** Usar TODAS as iniciais do nome (obrigatórias)
- **PRIORIDADE 2:** Intercalar com dígitos nos espaços restantes
- Posição ímpar: Inicial do nome (1ª, 3ª, 5ª, 7ª posição)
- Posição par: Dígito de preenchimento (2ª, 4ª, 6ª, 8ª posição)

### Ordem de Prioridade dos Dígitos
1. CEP: Usar os 3 últimos dígitos primeiro
2. Telefone: Usar os 3 últimos dígitos em seguida

### Tratamento de Casos Especiais
- **Iniciais obrigatórias:** TODAS as iniciais devem ser incluídas no ID
- Se houver mais iniciais que espaços ímpares, usar iniciais consecutivamente
- Dígitos são usados apenas para completar os 15 caracteres
- Se não houver dígitos suficientes, usar '0' como preenchimento
- Se CEP estiver vazio, pular para telefone

## CÓDIGOS DE CARGOS
- ADM. PEDAGÓGICO: _AP
- CDA: _CDA
- PROFESSORA: _PR
- PROFESSOR: _PR
- CONSULTOR COMERCIAL: _CC
- ADM. FINANCEIRO: _AF
- COORD. PEDAGÓGICA: _CP

## VALIDAÇÃO FINAL
- O ID final deve ter exatamente 15 caracteres
- Formato: EMP_XXXXXXXX_XX ou EMP_XXXXXXX_XXX
- Todos os caracteres alfabéticos devem estar em MAIÚSCULO
- Não deve conter espaços ou caracteres especiais (exceto underscore)

## BENEFÍCIOS DO SISTEMA

1. Padronização: Identificação consistente no banco de dados
2. Rastreabilidade: Informações do funcionário codificadas no próprio ID
3. Unicidade: Combinação de dados pessoais garante IDs únicos
4. Legibilidade: Estrutura clara e interpretável com intercalação organizada
5. Identificação de Cargo: Sufixo permite identificação rápida da função

Este sistema garante que cada funcionário tenha um identificador único, padronizado e
informativo, com a intercalação de iniciais e dígitos proporcionando maior
variabilidade e unicidade dos IDs gerados.
